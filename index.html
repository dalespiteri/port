About Me

Hi, I’m Dale, and I’m a designer.

I’ve spent the last ten years as a professional graphic designer. I love design, but working with a static canvas can start to feel a bit dull. I want to add functionality. What could suit a graphic designer looking to add function to form better than web design?

So I researched. I looked at post grad programs. I looked at boot camps. I looked at self teaching. I asked people in the field. Overwhelmingly, I was pushed to teach myself. Why? Because learning doesn’t stop. If you can’t teach yourself, you’ll eventually be phased out. So I did that. I taught myself to code. I’m still teaching myself to code. I’ll always be teaching myself because new and exciting tools will continue to be developed and at my disposal.

So now, I’m Dale, and I'm a web designer.

I actually first learned HTML as a kid. Making sites with friends using Geocities and Angelfire about videogames we used to play, I learned the basic building blocks of a webpage. It was a hobby for a couple years and it's amazing to see how much I had left unturned now that I am relearning it.

As someone who comes from a design background, CSS has been my favourite language so far to learn. The things you can do with it alone on top of very basic HTML structure is always a fun challenge for me. Seeing the introduction of Flex and Grids to CSS3 is something that fills me with confidence that I'll continue to enjoy learning the language as it develops and becomes more versatile.

On top of CSS, I am currently learning to use preprocessors to streamline my workflow. SASS and LESS are current skills I am adding to my toolbox.

jQuery is the most complicated language I have learned so far. This is actually why I love it. I enjoy challenges, and jQuery is a problem solver's dream. There is always a way to manipulate your page structure and style in fascinating ways, but at times, it takes a lot of creativity.

As a kid, I always enjoyed logic puzzles. You remember those right? You'd get a few paragraphs of a story, then a grid table with possibly a list of people, and a list of objects. You'd then have to use the information of the story to pair each person with an object using logical eliminations. jQuery reminds me of those. Sometimes you build from the front, sometimes you build from the back, sometimes you build from the middle. It's like building a Rube Goldberg contraption with language.

I am currently trying to supplement this skill with new frameworks such as React. New tools are just new options for the problems I'll inevitably be tasked with solving, so I can never have too many.

A website without images is just boring, but can also get fairly heavy. Working with large format images as a graphic designer has taught me a lot about compression and getting the most out of a small file size. This is very helpful in keeping graphic heavy sites light and fast on their feet.

Illustrator's vector software is second to none. It is ideal software for SVG file production, and iconography in web design. It keeps files small, and pixel perfect regardless of size.

On top of this, since starting web design, I've found Illustrator to be the ideal software for wire framing, and site planning. The way it uses shapes, layers, and colours offers much more versatility than Photoshop and make it much faster and user friendly for element design. Illustrator is by far my favourite tool in the Adobe tool kit and I have found it tremendously useful in my exploration of web design.
